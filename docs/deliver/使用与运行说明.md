# ResumeWise · 项目使用与运行说明

> 交付给使用方 / 助教 / 答辩老师的操作手册。  
> 源码根目录：`resume-web/src/`（静态原型，可完整演示）  
> 更新日期：2026-07-10

---

## 1. 项目是什么

ResumeWise 是面向**大学生求职**场景的 **AI 简历诊断、岗位匹配与优化演示系统**（大二小学期）。

核心闭环：

```text
输入简历 → 选择目标岗位（可参考样板简历）→ AI 诊断 → 查看问题 → 生成优化版本 → 导出
```

**不是**招聘平台：不做登录、不做爬虫、不接数据库、不自动投递。

## 4. 如何运行（演示模式 · 推荐答辩）

演示模式**不需要** API Key，也不需要代理，开箱可走完全流程。

### 4.1 方式 A：npm（推荐）

```bash
cd resume-web
npm install          # 首次如有依赖需要；当前主要为脚本入口
npm run dev
```

浏览器打开：

```text
http://127.0.0.1:5177
```

### 4.2 方式 B：Python 静态服务

```bash
cd resume-web/src
python -m http.server 5177
```

浏览器同样打开 `http://127.0.0.1:5177`。

### 4.3 确认模式

1. 打开任意页面，看顶栏。
2. 保持选中 **「演示模式 · Mock」**。
3. 无需点击「配置 API Key」。

## 5. 完整使用步骤（功能演示路径）

### 步骤 1 · 进入智能优化

1. 打开首页。
2. 点击 **「开始智能优化」**，进入填写页 `optimize.html`。

### 步骤 2 · 编辑简历（Markdown）

1. 左侧为 **我的简历**。
2. 默认为 **编辑模式**：可手写 Markdown，或整段粘贴 AI 生成的 Markdown。
3. 可点击 **「填入示例简历」** 快速填充演示内容。
4. 切换 **「渲染预览」** 查看排版效果。
5. 不会语法：点标题栏右侧 **「?」** 查看 Markdown 教学与最小示例。
6. 简历会缓存到浏览器 `localStorage`，刷新页面一般不丢。

### 步骤 3 · 选择目标岗位

1. 右侧 **目标岗位** 列表单选（5 个 Mock 岗位）。
2. **点击某一岗位卡片**后：
   - 该卡片会高亮；
   - **岗位要求摘要** 与 **岗位关键词** 展开在**该卡片内部下方**（不同技术栈关键词有独立配色）。
3. 需要对照范文时：点卡片右侧 **「样板简历参考」** 打开弹窗浏览；可将样板填入左侧或复制。

### 步骤 4 · 开始 AI 诊断

1. 确认简历非空、已选岗位。
2. 点击 **「开始 AI 诊断」**。
3. 进入诊断页，查看：
   - 综合评分（关键词匹配 × 40% + AI 内容 × 60%）
   - 岗位关键词覆盖（已匹配 / 建议补充）
   - 结构化问题与建议（按优先级）

### 步骤 5 · 生成优化版本并导出

1. 在诊断页点击生成优化版本（按页面按钮文案操作）。
2. 进入结果页：左侧原始 / 右侧优化对照。
3. 查看修改说明。
4. 导出：**MD / TXT / PDF / Word**（按页面提供的按钮）。

## 6. 真实模式 · DeepSeek（可选）

用于展示「真的调用了大模型」。答辩若网络或额度不稳定，**请优先使用演示模式**。

### 6.1 启动代理

**保持前端服务运行**，再开一个终端：

```bash
cd resume-web
npm run proxy
```

代理地址：`http://127.0.0.1:8787`（窗口不要关）。

### 6.2 配置 Key

1. 浏览器顶栏切换到 **「真实模式 · DeepSeek」**。
2. 点击 **「配置 API Key」**。
3. 粘贴你的 DeepSeek API Key。
4. 选择模型：
   - **V4 Flash**（`deepseek-v4-flash`）— 更快、成本更低
   - **V4 Pro**（`deepseek-v4-pro`）— 质量更高
5. 保存。Key **只存在本机浏览器**，不会写入 Git 仓库。

### 6.3 再走业务流程

与第 5 节相同。诊断与优化结果将由模型根据当前简历 + 岗位**动态生成**。

### 6.4 真实模式排查

| 现象 | 处理 |
|---|---|
| 请求失败 / 无响应 | 确认 `npm run proxy` 仍在运行 |
| 401 / 鉴权失败 | 检查 Key 是否正确、是否过期 |
| 超时 | 换 Flash 模型，或检查网络 |
| 答辩风险 | **立即切回演示模式 · Mock** |

## 7. 演示模式 vs 真实模式（对照）

| 对比项 | 演示模式 · Mock | 真实模式 · DeepSeek |
|---|---|---|
| API Key | 不需要 | 需要 |

| 诊断 / 优化内容 | 内置固定示例 | 模型实时生成 |
| 关键词匹配分 | 本地计算 | 本地计算（同一套） |
| AI 内容分 | 示例固定分 | 模型返回 |
| 推荐场景 | 答辩主路径 | 展示真实调用 |

页面内说明也可查看：**关于项目**（`about.html`）。

## 8. 评分公式（答辩可讲）

```text
综合评分 = 关键词匹配分 × 40% + AI 内容分 × 60%
```

- **关键词匹配分**：本地根据岗位关键词在简历中的覆盖比例计算，结果稳定、可解释。
- **AI 内容分**：模型对表达、项目、技能质量的评价（Mock 为固定示例分）。

详见仓库 `docs/adr/0001-scoring-formula.md`。

## 9. 本机数据说明（隐私）

| 存储位置 | 内容 |
|---|---|
| `localStorage` · `rw-mode` | 当前模式 |
| `localStorage` · `rw-deepseek-key` 等 | Key / 模型（仅本机） |
| `localStorage` · `rw-resume` / `rw-job` | 最近简历与岗位 |
| `sessionStorage` | 真实模式动态诊断 / 优化结果（关标签即清） |

清除方式：浏览器站点数据 / 应用内清空简历按钮。**请勿把带 Key 的截图提交到公开仓库。**

## 10. 常见问题

**Q：打开页面是空白或样式丢失？**  
A：请用本地 HTTP 服务打开（`npm run dev` 或 `python -m http.server`），不要直接双击用 `file://` 打开部分浏览器会拦截脚本。

**Q：点了诊断没反应？**  
A：检查是否填写了简历；真实模式下检查代理与 Key。

**Q：样板简历和真实简历有什么关系？**  
A：样板仅作「写法参考」，默认不会自动覆盖你的输入；可手动「填入样板」后再改。

**Q：后续 Vue / Spring Boot 呢？**  
A：当前交付为可演示的静态前端原型；业务后端与 Vue 复刻为规划项，见 `README.md` 与 `docs/contexts/`。
