# ResumeWise · 怎么用、怎么跑

> 帮你把项目在本机跑起来，走完一遍完整流程。  
> 代码在 `resume-web/src/`，更新于 2026-07-10。

---

## 这是个什么项目？

简单说：**帮大学生改简历的小工具**。

你把简历贴进去，选一个想投的岗位，系统会告诉你哪里写得不够好，再给你一版改好的对照着看，最后还能导出。

整条链路：

```text
贴简历 → 选岗位 → AI 诊断 → 看问题 → 生成优化版 → 导出
```

它不是招聘网站：没有登录、没有爬虫、没有数据库、也不会帮你自动投递。

---

## 你需要准备什么

- **浏览器**：Chrome / Edge / Firefox，新版本就行
- **Node.js**：建议 18+（用来 `npm run dev`）
- **Python**：3.9+（只有开「真实模式」时才需要；也可以纯用 Python 起静态服务）
- **网络**：演示模式不用联网；真实模式要能访问 DeepSeek

## 跑起来（推荐：演示模式）

演示模式**不用 API Key**，也不用开代理，答辩用这个最稳。

### 方法一：npm（推荐）

```bash
cd resume-web
npm install    # 第一次可能需要
npm run dev
```

浏览器打开 `http://127.0.0.1:5177`。

### 方法二：Python

```bash
cd resume-web/src
python -m http.server 5177
```

同样打开 `http://127.0.0.1:5177`。

### 确认一下模式

看页面顶栏，保持 **「演示模式 · Mock」** 就行，不用点「配置 API Key」。

## 完整走一遍（演示路径）

### 1. 进填写页

首页点 **「开始智能优化」**，进入 `optimize.html`。

### 2. 填简历

左边是简历区，支持 **Markdown** 编辑和预览：

- 可以直接手写，也可以把 AI 生成的 Markdown 整段贴进来
- 懒得写就点 **「填入示例简历」**
- 想看点效果就切 **「渲染预览」**
- 不会 Markdown？点标题栏右边的 **「?」**，有最简教学
- 内容会自动存到浏览器，刷新一般还在

### 3. 选岗位

右边 5 个岗位卡片，**点一下就能选中**：

- 选中后卡片会高亮
- 卡片下方会展开**岗位要求**和**关键词**（不同技术栈颜色不一样）
- 想看范文？点 **「样板简历参考」** 弹窗看看，可以复制或填到左边

### 4. 开始诊断

简历不为空、岗位也选好了，点 **「开始 AI 诊断」**。

诊断页你会看到：

- **综合评分**（下面会拆成关键词分 + AI 内容分）
- **关键词覆盖**：哪些已经有了、哪些还缺
- **问题列表**：每条有问题说明和改法建议

### 5. 生成优化版 & 导出

诊断页点生成优化，进结果页：

- 左边原始简历，右边优化后的
- 下面有修改说明
- 可以导出 **MD / TXT / PDF / Word**

## 真实模式（可选，想展示真 AI 再用）

网络不稳或答辩时间紧的话，**老老实实走演示模式就好**。

### 怎么开

前端服务别关，再开一个终端：

```bash
cd resume-web
npm run proxy    # 代理跑在 http://127.0.0.1:8787，窗口别关
```

然后浏览器里：

1. 顶栏切到 **「真实模式 · DeepSeek」**
2. 点 **「配置 API Key」**，贴上你的 Key
3. 选模型：Flash 快一点便宜一点，Pro 质量高一点
4. Key **只存在你这台电脑的浏览器里**，不会进 Git

后面的操作跟上面第 5 节一样，只是诊断和优化内容是模型实时生成的。

### 出问题了怎么办

| 情况 | 试试这个 |
|---|---|
| 点了没反应 | 代理还在跑吗？Key 填对了吗？ |
| 401 鉴权失败 | Key 过期或抄错了 |
| 一直转圈 | 换 Flash 模型，或检查网络 |
| 答辩现场翻车 | **立刻切回演示模式 · Mock** |

## 两种模式有啥区别

| | 演示模式 · Mock | 真实模式 · DeepSeek |
|---|---|---|
| 要 Key 吗 | 不要 | 要 |
| 要代理吗 | 不要 | 要（8787 端口） |
| 诊断内容 | 内置好的示例 | 模型实时生成 |
| 关键词分 | 本地算 | 本地算（同一套） |
| AI 内容分 | 固定示例分 | 模型给 |
| 适合啥场景 | **答辩主路径** | 加分展示 |

更详细的对照表在 **关于项目** 页（`about.html`）里也有。

---

## 评分怎么算的（答辩可能会问）

```text
综合评分 = 关键词匹配分 × 40% + AI 内容分 × 60%
```

- **关键词匹配分**：看你简历里有没有岗位要的那些词，本地算，每次一样，好解释
- **AI 内容分**：模型评价你写得怎么样（演示模式里是固定示例分）

想深挖可以看 `docs/adr/0001-scoring-formula.md`。

## 数据存在哪（隐私相关）

| 位置 | 存啥 |
|---|---|
| `localStorage` | 当前模式、Key（如有）、最近简历和岗位 |
| `sessionStorage` | 真实模式下的诊断/优化结果（关标签就没了） |

想清空：浏览器清站点数据，或者页面里点清空简历。**带 Key 的截图别往公开仓库扔。**

## 常见问题

**页面空白 / 样式没了？**  
别直接双击 html 用 `file://` 打开，用 `npm run dev` 或 `python -m http.server` 起本地服务。

**点了诊断没动静？**  
先看简历填了没；真实模式再看代理和 Key。

**样板简历和我的简历啥关系？**  
样板只是给你参考写法，不会自动覆盖你写的内容，要的话手动填进去再改。

**后面还会做 Vue / Spring Boot 吗？**  
现在是能完整演示的静态前端；Vue + 后端是规划里的下一步，见 `README.md`。
