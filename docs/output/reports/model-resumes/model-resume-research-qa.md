# 样板简历调研 · QA 清单

> 使用方式：把本文 + `model-resume-research-brief.md` 一并丢给网页 GPT / Grok。  
> 要求：逐题回答；能附样本的题目请直接附 **脱敏纯文本** 或 **可下载链接 + 抽取正文**。

---

## A. 信息源与版权

| # | 问题 | 期望输出 |
|---|---|---|
| A1 | 国内「大学生实习简历范文」常见靠谱公开来源有哪些？（教务/求职站点/开源仓库） | 列表：名称、URL、可信度、是否允许教学演示 |
| A2 | 哪些来源明确禁止二次分发？哪些适合改写后作 Mock？ | 表格 |
| A3 | 英文 resume 样本是否有必要？本项目默认中文场景下如何取舍？ | 结论 + 理由 |
| A4 | PDF 样板如何批量变成纯文本最省事？（工具链建议即可） | 步骤，不要求你方执行 |

---

## B. 分岗位样板（核心交付）

对 **每一个** jobId（`java` / `frontend` / `test` / `product` / `data`）回答 B1–B6：

| # | 问题 | 期望输出 |
|---|---|---|
| B1 | 该岗位实习 JD 里最高频的 8–12 个关键词是什么？ | 中英文混排列表 |
| B2 | 一封「合格但普通」的实习简历长什么样？ | 完整脱敏纯文本 + 点评 3 条弱点 |
| B3 | 一封「值得当模范」的实习简历长什么样？ | 完整脱敏纯文本 + 点评 3 条优点 |
| B4 | 模范简历相对普通简历，在 **项目描述** 上差在哪？ | 对照表（差 → 好 示例句） |
| B5 | 该岗位最常见的 5 类简历问题是什么？ | 对应 Issue Type：表达/项目/技能/其他 |
| B6 | 若只能保留 1 份模范简历进系统，选哪份？为什么？ | 选定正文 + 理由 |

## C. 结构与字段（服务 Agent）

| # | 问题 | 期望输出 |
|---|---|---|
| C1 | 样板简历是否必须分「教育 / 技能 / 项目 / 校园」固定区块？有无更贴实习场景的结构？ | 推荐结构模板 |
| C2 | 给大模型做「对照诊断」时，Prompt 里应如何引用样板？（摘要 vs 全文） | 建议策略与字数 |
| C3 | 关键词匹配分只靠关键词列表够不够？是否需要从样板中自动抽关键词？ | 利弊 + 推荐 |
| C4 | 演示模式（Mock）下，样板简历如何参与「假诊断」结果生成才显得真实？ | 规则建议 |
| C5 | 真实模式（DeepSeek 等）下，样板简历与用户简历、岗位 JD 的输入优先级？ | 排序说明 |

## D. 双模式与配置（产品）

| # | 问题 | 期望输出 |
|---|---|---|
| D1 | 演示模式 vs 真实 API 模式，样板简历数据是否完全同一份？ | 是/否 + 例外 |
| D2 | DeepSeek 类文本模型做简历诊断的推荐参数？（温度、max_tokens 量级） | 经验值区间 |
| D3 | API Key 应出现在前端设置里还是仅后端？小学期演示如何折中？ | 安全结论（本项目约束：生产仅后端） |
| D4 | 无样板简历时系统是否允许开真实模式？ | 推荐门禁策略 |

---

## E. 交付清单自检

调研方交付前请自检：

- [ ] 5 个岗位均有 ≥1 份模范纯文本
- [ ] 每份有来源与脱敏说明
- [ ] 每岗有关键词列表（可与模范正文对照）
- [ ] 文件命名可映射 `jobId`
- [ ] 无真实手机号/身份证/学校隐私超范围信息
- [ ] 附一份 `index`（表格或 JSON）总览
