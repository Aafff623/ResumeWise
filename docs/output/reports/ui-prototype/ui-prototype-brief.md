# ResumeWise 前端界面原型出图 Brief

> **用途**：把每个页面的功能点、按钮文案、数据区块、交互状态写清楚，避免「生成一个简历优化界面」这类模糊 prompt 导致漏功能、错流程。

## 0. MVP 功能清单（出图必须覆盖）

以下能力来自产品域文档，**每张原型图都必须能在画面上找到对应视觉元素**：

| # | 功能 | 界面落点 | 必现元素 |
|---|---|---|---|
| F1 | 简历输入（粘贴纯文本） | 简历优化页 | 大文本框、字数提示、清空按钮 |
| F2 | 一键填充示例简历 | 简历优化页 | 「使用示例简历」按钮（至少 1 个） |
| F3 | 目标岗位选择 | 简历优化页 | 5 个 Mock 岗位卡片，单选高亮 |
| F4 | 发起 AI 诊断 | 简历优化页 → 诊断页 | 主 CTA「AI 诊断」；未选岗位/空简历时禁用态 |
| F5 | 综合评分展示 | 诊断页 | 大号分数环/数字 + 公式说明「关键词 40% + AI 内容 60%」 |
| F6 | 匹配度 / 分项分 | 诊断页 | 匹配度进度条；关键词匹配分、AI 内容分两列小卡 |
| F7 | 关键词覆盖 | 诊断页 | 已匹配（绿色 Tag）、缺失（红色 Tag）两组 |
| F8 | 结构化诊断问题 | 诊断页 | 问题列表：类型徽章 + 描述 + 优化建议 |
| F9 | 诊断摘要 | 诊断页 | AI 一段话总结（`summary` 字段） |
| F10 | 生成优化版本 | 诊断页 → 结果页 | 次 CTA「生成优化版本」 |
| F11 | 原始 ↔ 优化对照 | 结果页 | 左右双栏（移动端上下），标题「原始简历」「AI 优化简历」 |
| F12 | 修改说明 | 结果页 | 编号列表 `changes[]` |
| F13 | 复制 / 导出 | 结果页 | 「复制优化结果」「导出 .txt」按钮 |
| F14 | 本地缓存提示 | 简历优化页 | 轻提示「已自动保存到本地，刷新不丢失」 |
| F15 | 请求态 | 诊断页 / 结果页 | Loading 遮罩或骨架屏 + 文案「AI 分析中…」 |
| F16 | 演示鲁棒 | 全局角标 | 小标签「AI Mock 模式」或「演示模式」 |

**Mock 岗位（卡片上必须出现的中文标题）**：

1. Java 后端开发实习生
2. 前端开发实习生（Vue）
3. 软件测试实习生
4. 产品经理实习生
5. 数据分析实习生

**不做进画面**（避免 GPT 乱加）：登录注册、文件上传 PDF/Word、聊天对话框、招聘列表爬虫、管理后台。
