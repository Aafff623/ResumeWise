项目定位

面向大学生求职场景的 AI 简历诊断、岗位匹配与优化演示系统（大二小学期）。核心闭环：

输入简历 → 选择目标岗位（+ 样板参照）→ AI 诊断 → 生成优化版本 → 导出

不是招聘平台；不做登录、爬虫、数据库、企业端、自动投递。

源码在哪（重要）

前端规范 SRC 目录：

resume-web/src/

  • index.html 首页 landing
  • optimize.html 填写简历 / 岗位 / 样板
  • review.html AI 诊断结果
  • result.html 优化对照与导出
  • assets/js/ 交互 · DeepSeek 调用 · 岗位数据
  • deepseek-proxy.py 真实模式本机 CORS 代理

docs/ 放规范、ADR、配图、PRD；不要在 docs/.../html-src 里继续开发（仅迁移指针）。

演示模式 vs 真实模式（维护必读）

顶栏可随时切换。两种模式页面流程相同（填简历 → 选岗 → 诊断 → 优化 → 导出），差别只在「AI 结果从哪来」。

演示模式 · Mock
  • 要 Key 吗？ 不需要
  • 要开代理吗？ 不需要，只开 npm run dev
  • 结果从哪来？ 前端内置示例诊断 / 优化文案（固定内容）
  • 换简历会变吗？ 基本不变——适合答辩「保证能跑通」
  • 适合场景 课堂演示、断网、无额度、助教验收
  • 如何切换 任意页顶栏 →「演示模式 · Mock」
真实模式 · DeepSeek
  • 要 Key 吗？ 要。仅存在本机浏览器 localStorage
  • 要开代理吗？ 要：另开终端 npm run proxy（:8787，绕过 CORS）
  • 模型 V4 Flash（deepseek-v4-flash）/ V4 Pro（deepseek-v4-pro）
  • 结果从哪来？ 模型按「简历 + 岗位」动态生成，再写入 sessionStorage 渲染
  • 换简历会变吗？ 会——内容随输入变化，适合展示「真 AI」
  • 安全 Key 不进 Git；别人 clone 仓库拿不到你的 Key

对比项 演示模式 · Mock 真实模式 · DeepSeek
API Key 不需要 需要（本机配置）
本机代理 :8787 不需要 必须开着
诊断 / 优化内容 内置固定示例 模型实时生成

真实模式启用步骤
  1. 终端 1：cd resume-web && npm run dev（:5177）
  2. 终端 2：cd resume-web && npm run proxy（:8787，勿关）
  3. 浏览器顶栏切到「真实模式 · DeepSeek」
  4. 点「配置 API Key」填入 Key，选 Flash 或 Pro，保存
  5. 走智能诊断流程；结果由模型返回

常见问题： 真实模式无响应 → 先看代理是否在跑、Key 是否有效、额度是否够；答辩不稳时立刻切回演示模式。

本机存储键（改代码时对照）
  • rw-mode：当前模式（mock / live）
  • rw-deepseek-key · rw-deepseek-model：真实模式配置
  • rw-resume · rw-job：最近简历与岗位
  • 诊断 / 优化动态结果：主要用 sessionStorage（关闭标签页即清）

相关实现：assets/js/app.js（模式切换）· assets/js/deepseek-api.js（调用）· deepseek-proxy.py（代理）。

评分公式（答辩必背）

综合评分 = 关键词匹配分 × 40% + AI 内容分 × 60%

关键词分：本地按岗位 keywords 在简历中命中比例计算。内容分：Mock 固定 / 真实模式由 DeepSeek 给出。见 ADR-0001。

本地启动（维护速查）

cd resume-web npm run dev # 前端 :5177 npm run proxy # 真实模式代理 :8787

  • 演示：只开 npm run dev 即可
  • 真实：两个终端都开 + 配置 Key
  • 改界面：改 resume-web/src/*.html 与 assets/
范围边界
  • 不做：登录注册、数据库、招聘爬虫、PDF/Word 上传解析、企业端
  • 不做：多轮聊天 Agent、自动投递
  • 当前前端为静态原型；Vue3 + Spring Boot 业务复刻为后续阶段
  • 生产环境应将 Key 收到后端（ADR-0003），勿长期依赖浏览器直连
