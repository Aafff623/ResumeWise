项目定位

面向大学生求职场景的 AI 简历诊断、岗位匹配与优化演示系统（大二小学期）。核心闭环：

输入简历 → 选择目标岗位（+ 样板参照）→ AI 诊断 → 生成优化版本 → 导出

不是招聘平台；不做登录、爬虫、数据库、企业端、自动投递。

源码在哪（重要）

前端规范 SRC 目录：

resume-web/src/

  • index.html 首页 landing
  • optimize.html 填写简历 / 岗位 / 样板
  • review.html AI 诊断结果
  • result.html 优化对照与导出
  • assets/js/ 交互 · DeepSeek 调用 · 岗位数据
  • deepseek-proxy.py 真实模式本机 CORS 代理

docs/ 放规范、ADR、配图、PRD；不要在 docs/.../html-src 里继续开发（仅迁移指针）。

演示模式 vs 真实模式
演示模式 · Mock
  • 要 Key 吗？ 不需要
  • 结果从哪来？ 页面内置示例数据
  • 适合场景 答辩、无网、无额度
  • 如何切换 顶栏点「演示模式 · Mock」
真实模式 · DeepSeek
  • 要 Key 吗？ 要，存在本机 localStorage
  • 模型 V4 Flash / V4 Pro（id：deepseek-v4-flash / deepseek-v4-pro）
  • 还要开什么 npm run proxy（:8787）
  • 结果从哪来？ 模型动态返回，写入 sessionStorage 再渲染
  • 安全 Key 不进 Git；别人 clone 拿不到你的 Key
评分公式（答辩必背）

综合评分 = 关键词匹配分 × 40% + AI 内容分 × 60%

关键词分：本地按岗位 keywords 在简历中命中比例计算。内容分：Mock 固定 / 真实模式由 DeepSeek 给出。见 ADR-0001。

本地启动（维护速查）

cd resume-web npm run dev # 前端 :5177 npm run proxy # 真实模式代理 :8787

  • 演示：只开 npm run dev 即可
  • 真实：两个终端都开 + 配置 Key
  • 改界面：改 resume-web/src/*.html 与 assets/
范围边界
  • 不做：登录注册、数据库、招聘爬虫、PDF/Word 上传解析、企业端
  • 不做：多轮聊天 Agent、自动投递
  • 当前前端为静态原型；Vue3 + Spring Boot 业务复刻为后续阶段
  • 生产环境应将 Key 收到后端（ADR-0003），勿长期依赖浏览器直连
